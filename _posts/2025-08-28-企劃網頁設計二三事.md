# 企劃網頁設計二三事

## 前言

IN N 百年 進行中的 TEAM 最近終於有名字了，也開始了新企畫，因為做了好久的東西真的很想要盡善盡美，所以打算能做得盡量去做，想想我現在有的能力(ㄟ?)，盡可能嘗試。  
老實說我不是很擅長做設計，接觸的少，也沒有特別有系統地學過，BUT 身為資工系學生......要說會做，也是會做，至少架構該懂得有懂。  !!!前端設計還是讓我很痛苦！！

## 設計發想
我基本上會先思考的內容是[網頁要呈現的資訊]，比起美麗酷炫的設計，我其實更在意大家是否能直觀輕鬆的獲取自己需要的資訊，在之後才是其他的使用者體驗，是否足夠有沉浸感、新鮮且讓人眼睛為之一亮。

如果好奇我的設計可以到 [White House](https://white-house-delta.vercel.app) 看看 ... 但應該就是中規中矩的內容 目前來說 XDD

因為是 for 企劃用 還是有稍微想一下要融入主題和玩法，所以越想越複雜：
- 從 企劃宣傳網站(單純靜態網站) 
- 到 宣傳+QA+公告(那就得連資料庫拉，大家都不太熟悉其他資料庫還是用試算表好了)
- 主要原因還是單純鑲入試算表對於使用者體驗真的稱不上友好(o0O老師知道我考慮那麼多會感動嗎?) 所以打算做簡單的方面呈現的網站

但!我又想到企畫的內容玩法，資訊呈現...以我之前的經驗來說，主辦光是要梳理個別成員、小組、陣營資訊就要花很多時間了，那既然臥都做網站了 也串接資料庫了...?

於是擴展為：
- 宣傳+QA+公告+個人資訊頁(包含所持有物品、主支線任務、數值等資訊 只要登入就能輕鬆看到喔!!)

然後解決衍伸問題：
- 要在 Discord 用機器人玩+處理角色色數值
- 把開源機器人 hktrpg 改造成專門 for 企劃的擲骰+遊戲機器人
- 網站設計也從宣傳+解釋官網變成企劃世界觀特定組織的宣傳官網

更多功能設計：
- 事件系統
- GM管理系統

## 架構說明

整個專案採用現代化的前端技術棧：

1. 前端框架與語言
   - JavaScript (83.5%)：主要開發語言
   - TypeScript (1.7%)：用於添加型別檢查
   - CSS (12.8%) 與 SCSS (1.4%)：負責樣式與排版
   - HTML (0.5%)：基礎結構

2. 部署環境
   - 使用 Vercel 進行部署
   - Docker 容器化 (0.1%) 確保開發環境一致性

3. 主要功能模組
   - 用戶認證系統
   - 個人資訊頁面
   - 任務追蹤系統
   - Discord 機器人整合
   - GM 管理介面
   - 事件系統

4. 資料儲存
   - 使用試算表作為輕量級資料庫
   - Discord Bot 資料同步機制

## 技術重點

1. 前端架構
   - 響應式設計，確保多裝置支援
   - 模組化元件設計
   - 狀態管理系統

2. 整合服務
   - Discord Bot API 整合
   - 試算表 API 串接
   - 用戶驗證系統

3. 效能優化
   - 代碼分割
   - 靜態資源優化
   - 快取策略

4. 開發工具鏈
   - ESLint 程式碼檢查
   - Prettier 程式碼格式化
   - TypeScript 型別檢查
   - Docker 容器化開發環境
